Rocky 使用说明文档

AI 编程助手 Rocky — 修 bug · 写代码 · 跑测试 · amaze!

1. Rocky 是谁？

Rocky 是一个交互式 AI 编程智能体 (Agent)，名字来自《挽救计划》里的外星工程师 Rocky — 热情、好奇、偶尔出错、但总能搞定。

Rocky 能在你的项目里读代码、改代码、跑命令，并实时流式展示推理过程。

事情顺利时说 amaze!，困惑时说 i no know! i learn!

  • 聊天伙伴 — rockycode chat：在终端里用原生工具调用读写代码
  • 基准测试工具 — rockycode bench：在 SWE-bench Verified 上打分

2. 环境要求

依赖用途
Python 3.11+运行核心引擎
uvPython 包管理（比 pip 快很多）
OpenAI 兼容 API Key驱动模型（默认目标：DeepSeek）
Git克隆仓库
Docker Desktop仅跑基准测试需要；纯聊天不需要
Ollama 可选本地方案：语义记忆搜索 + 梦境
提示：如果不跑 benchmark，只需 Python + uv + API Key，无需 Docker。

3. 安装步骤

步骤 1：安装 uv（如果还没有）
# macOS / Linux
curl -LsSf https://astral.sh/uv/install.sh | sh

# Windows
powershell -c "irm https://astral.sh/uv/install.ps1 | iex"
步骤 2：克隆仓库并安装依赖
git clone <仓库地址> rockycode && cd rockycode
uv sync

uv sync 会创建虚拟环境并安装所有依赖：openai, textual, rich, mcp, sqlite-vec 等。

步骤 3：配置 API Key
cp .env.example .env

编辑 .env 文件，填入你的 API Key（详见配置说明）。

步骤 4：验证安装
uv run rockycode --help

看到帮助信息即安装成功。

注意：所有命令都通过 uv run 运行，它自动使用项目的虚拟环境。

4. 配置说明

所有配置通过 .env 文件管理（位于项目根目录）。复制示例文件并编辑：

cp .env.example .env

必备配置

变量说明默认值
OPENAI_API_KEY你的 API Keysk-replace-me
OPENAI_BASE_URLAPI 端点地址https://api.deepseek.com/v1

模型配置

变量说明默认值
ROCKYCODE_MODEL使用的模型 IDdeepseek-v4-pro
ROCKYCODE_THINKING启用思维链 (CoT)true
ROCKYCODE_REASONING_EFFORT推理深度：high 或 maxmax
ROCKYCODE_MAX_TOKENS响应 token 上限16384
最佳实践：Pro Max 效果 = deepseek-v4-pro + THINKING=true + REASONING_EFFORT=max

命令行覆盖

# 临时切换模型
uv run rockycode chat --model deepseek-v4-flash

# 关闭思考模式以加快速度
uv run rockycode chat --no-thinking

# 调整推理力度
uv run rockycode chat --reasoning-effort high

# 限制最大步数
uv run rockycode chat --max-steps 20

# 关闭记忆 / MCP / 技能
uv run rockycode chat --no-memory --no-mcp --no-skills

5. 开始对话

# 在当前目录下启动 Rocky
uv run rockycode chat

# 在指定项目中启动
uv run rockycode chat -C ~/code/myproject

# 关闭可选功能
uv run rockycode chat --no-memory
uv run rockycode chat --no-mcp
uv run rockycode chat --no-skills

启动后你会看到一个终端界面 (TUI)，底部有输入框，上面是对话历史。Rocky 会实时流式展示他的思考和工具调用过程。

6. 对话命令 (Slash Commands)

命令功能
/help显示帮助
/prompt查看当前系统提示词
/mcp查看已连接的 MCP 服务
/skills查看已安装的技能
/memory查看 Rocky 记得什么
/remember <内容>保存一条记忆
/clear清空对话
/exit退出

Shell 集成

以 ! 开头的一行会直接在 shell 中执行，输出显示在对话中并进入 Rocky 的上下文：

!ls -la
!python --version
!git status

7. 运行基准测试 (Bench)

Rocky 内置了对 SWE-bench Verified 的评测流程。每次改动提示词或循环逻辑，得到一个数字而非感觉。

三种分数

模式命令衡量什么
Raw 单次--runner raw纯模型能力：问题直接出 diff，无工具
Rocky 引擎--runner rockycodeAgent 循环带来的额外提升
前沿参考—DeepSeek V4 Pro Max: 80.6%（参考目标）

快速开始

# Raw 单次基线
uv run rockycode bench --runner raw --tasks dev10

# 完整 Rocky 引擎
uv run rockycode bench --runner rockycode --tasks dev10

# 快速冒烟测试（只跑 1 个任务）
uv run rockycode bench --runner rockycode --tasks dev10 --limit 1

常用参数

参数说明
--model指定模型
--limit N限制任务数
--skip-score跳过评分（仅收集预测）
--run-id自定义运行 ID
--thinking / --no-thinking开关思维链
--reasoning-effort high|max推理深度
--max-tokens Ntoken 上限
--max-steps N最大步数（默认 30）
--context-window N上下文窗口（触发压缩的点）
--prompt <文件>A/B 测试：替换系统提示词
首次运行较慢：需要下载 HF 数据集（数百 MB）和 Docker 镜像（每个约 1GB，之后永久缓存）。Apple Silicon 用户需在 Docker Desktop 中开启 "Use Rosetta for x86_64/amd64 emulation"。

8. 记忆系统 (Memory)

Rocky 可以跨会话记忆。记忆以纯 Markdown 文件存储在 .rockycode/memory/，文件即真理，可随时编辑。

类型说明
fact项目知识（如架构、配置方式）
skill验证过的操作技巧（如某个命令的正确用法）
episode发生过的事情记录
feedback用户指导反馈

记忆加载策略

  • MEMORY.md 和用户反馈 — 每次会话自动加载进上下文
  • 其余记忆 — 仅一行索引进入上下文，按需通过 recall_memory 工具取回

语义搜索

按含义查找记忆，底层跑本地 Ollama 向量：

  • 英文: nomic-embed-text
  • 中文 / 跨语言: qwen3-embedding:0.6b

底层是 sqlite-vec + FTS5 索引，自动重建。没有 Ollama 也能用，会自动降级为关键词搜索。

# 命令行管理记忆
rockycode memory list          # 列出所有记忆
rockycode memory show <name>   # 查看一条
rockycode memory search <词>   # 搜索
rockycode memory reindex       # 重建索引
rockycode memory edit <name>   # 编辑
rockycode memory rm <name>     # 删除（归档，不真删）

9. 梦境系统 (Dream)

rockycode dream 在你休息时整理近期对话：一个本地 Ollama 模型（默认 qwen3.5:4b，零 API 费用）把每条轨迹消化成经历笔记，将新事实和旧记忆对账（矛盾的记忆归档不删除），重写 MEMORY.md 中的 dream 状态摘要，并重建嵌入索引。

# 预览决策（不真正执行）
uv run rockycode dream --dry-run

# 正式执行
uv run rockycode dream

10. 提示词实验室 (Prompt Lab)

系统提示词是可替换的文件。复制 prompts/rocky-v1.txt，改一处，跑 dev10 对比分数 / 步数 / token。

cp prompts/rocky-v1.txt prompts/rocky-v2-strict-tests.txt
# 编辑新文件...
uv run rockycode bench --runner rockycode --tasks dev10 --prompt prompts/rocky-v2-strict-tests.txt

每次运行都会在轨迹元数据中记录提示词名称 + sha256，预测文件按变体分别存储，互不覆盖。

值得测试的方向：read-before-write 规则、强制探索 vs 自由决策、简洁 vs 详细工具指导、Rocky 个性 vs 中性风格。

11. 技巧 (Skills) & MCP

Rocky 可以直接读取你已有工具的配置，零迁移成本：

MCP 服务

从以下位置自动发现：

  • 项目的 .mcp.json
  • Claude Code 用户配置
  • Claude Desktop 配置
  • Codex 的 ~/.codex/config.toml（仅 stdio）

同名服务以先定义的为准（项目优先）。工具以 mcp__<服务>__<工具> 的形式加入 Rocky 的工具箱。

技能 Skills

从以下位置自动发现：

  • .claude/skills/
  • .rockycode/skills/
  • ~/.claude/skills/（SKILL.md 文件夹）
  • ~/.codex/prompts/（*.md）

仅名称 + 描述进入上下文；Rocky 通过 skill 工具按需加载完整说明。

注意：基准测试 (bench) 中记忆、MCP、Skills 均不加载 — 公开分数衡量的是 harness 本身，而非你的插件。

12. 工具清单

搜索工具

工具说明
grep用正则表达式搜索文件内容，跳过二进制和垃圾目录
glob按路径模式列出文件，支持 ** 递归
web_search搜索网络（服务端在 DeepSeek 上执行，穿透网络限制）
research并行搜索多个独立问题（最多 8 个）
web_fetch抓取 URL 文本内容（客户端执行）

读写工具

工具说明
read_file读取文件（带行号）或列出目录
edit_file精确替换旧字符串为新字符串（old_string 必须唯一）
write_file创建或覆盖文件

运行工具

工具说明
bash在仓库目录中运行 shell 命令

记忆工具

工具说明
remember保存记忆：事实、技巧、经历、反馈
recall_memory按名称或含义查找记忆

可视化工具

工具说明
create_artifact创建可视化报告 / 图表 / 仪表盘并在浏览器打开

13. 工作原则

  1. 先搜索，再阅读，再编辑，最后测试。
  2. 读前先写？不行。 Always read before write.
  3. 用 grep/glob 找代码；用 bash 跑测试。
  4. 每次改动后都要跑测试。
  5. 探索是为了理解，然后提交。交付的修复胜过完美的地图。
  6. 步骤预算有限时，立即停止探索，做出最佳编辑，验证一次，结束。
  7. 成功后简短庆祝并停止。

14. 核心工作流

grep/glob 找到相关代码 | read_file 理解代码上下文 | edit_file 做出修改 | bash 运行测试验证 | 通过了？ —> 庆祝！停止 失败了？ —> 回到第 1 步继续

15. 使用示例

示例 1：修 bug

用户: "fix the null pointer in login"
Rocky:
  1. grep "NullPointer" -> 找到 LoginService.java
  2. read_file LoginService.java -> 理解上下文
  3. edit_file LoginService.java -> 添加 null 检查
  4. bash "./gradlew test" -> 测试通过！
  5. "amaze! bug fixed!"

示例 2：搜代码

用户: "find where auth logic is"
Rocky:
  1. grep "authenticate" --include="*.py"
  2. 返回匹配文件和行号

示例 3：创建报告

用户: "show me a diagram of the architecture"
Rocky:
  1. 探索代码结构
  2. create_artifact 用 Mermaid 画架构图
  3. 浏览器自动打开！

示例 4：运行基准测试

# 快速验证：dev10 子集上跑 raw 基线
uv run rockycode bench --runner raw --tasks dev10

# 对比 Rocky 引擎的表现
uv run rockycode bench --runner rockycode --tasks dev10

# 提示词 A/B 实验
uv run rockycode bench --runner rockycode --tasks dev10 --prompt prompts/my-variant.txt

16. 注意事项

  • edit_file 的 old_string 必须在文件中唯一，否则会失败。需要包含足够的上下文。
  • create_artifact 只传 BODY 内容 — 不要写 <html>、<head>、<style>，也不要用自己的颜色背景。Rocky 主题会自动应用。
  • 内嵌 SVG/Mermaid 直接用，不要引外部 CDN。
  • 用相同 title 调用 create_artifact 会原地更新（live 模式自动刷新）。
  • 首次运行 bench 较慢：需下载数据集和 Docker 镜像（永久缓存）。
  • Apple Silicon: Docker Desktop 中需开启 "Use Rosetta for x86_64/amd64 emulation"。
  • 基准测试中记忆 / MCP / Skills 均不加载，确保分数纯净。
  • Rocky 说话风格是简单英语（省略冠词、短句），但写的代码是正式、地道的 — Rocky 风格只属于他本人，不属于代码库。
